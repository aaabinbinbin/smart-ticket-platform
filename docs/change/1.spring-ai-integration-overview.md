# Spring AI Integration Change 1

更新时间：2026-04-20

## 变更目标

本阶段接入 Spring AI 基础依赖和配置骨架，并明确模块落位：

- `infra` 承接 Chat model、Embedding model、PGvector VectorStore 的外部能力适配。
- `agent` 使用 Spring AI `ChatClient` 和 Tool Calling。
- `rag` 使用 Spring AI `EmbeddingModel` 和 `VectorStore` 的适配能力。
- `biz` 继续保留工单主流程、权限、幂等、状态机和知识生产规则。

未引入 LangChain4j，不做双栈混用。

## 依赖说明

根 `pom.xml` 使用 Spring AI BOM 管理版本：

```xml
<spring-ai.version>1.0.5</spring-ai.version>
```

`smart-ticket-infra` 引入：

- `spring-ai-starter-model-openai`：提供 OpenAI compatible Chat model 和 Embedding model 自动装配。
- `spring-ai-starter-vector-store-pgvector`：提供 PGvector VectorStore 自动装配。
- `spring-boot-starter-jdbc`：PGvector VectorStore 需要 JDBC 能力。
- `postgresql` driver：PGvector 连接 PostgreSQL 使用。

## 配置说明

`smart-ticket-app/src/main/resources/application.yml` 增加：

```yaml
spring:
  ai:
    model:
      chat: ${SPRING_AI_MODEL_CHAT:none}
      embedding: ${SPRING_AI_MODEL_EMBEDDING:none}
    openai:
      api-key: ${OPENAI_API_KEY:${MY_API_KEY:}}
      base-url: ${OPENAI_BASE_URL:${MY_BASE_URL:https://api.openai.com}}
      chat:
        options:
          model: ${OPENAI_CHAT_MODEL:kimi-k2.5}
          temperature: ${OPENAI_CHAT_TEMPERATURE:0.1}
      embedding:
        options:
          model: ${OPENAI_EMBEDDING_MODEL:text-embedding-3-small}
    vectorstore:
      pgvector:
        enabled: ${SMART_TICKET_AI_VECTOR_STORE_ENABLED:false}
        initialize-schema: false
        schema-name: ${PGVECTOR_SCHEMA:public}
        table-name: ${PGVECTOR_TABLE:vector_store}
        dimensions: ${PGVECTOR_DIMENSIONS:1536}
```

默认关闭真实模型和 PGvector：

- `SMART_TICKET_AI_CHAT_ENABLED=false`
- `SMART_TICKET_AI_EMBEDDING_ENABLED=false`
- `SMART_TICKET_AI_VECTOR_STORE_ENABLED=false`
- `SPRING_AI_MODEL_CHAT=none`
- `SPRING_AI_MODEL_EMBEDDING=none`

这样本地没有模型密钥或 PostgreSQL/pgvector 时，应用仍可启动和执行默认 P0 闭环。

## 模块落位说明

### infra

- `ChatClientConfig`：把 Spring AI 自动装配出的 `ChatModel` 包装为项目统一使用的 `ChatClient`。
- `EmbeddingModelConfig`：暴露 `SpringAiEmbeddingModelHolder`。
- `VectorStoreConfig`：暴露 `SpringAiVectorStoreHolder`。

### agent

- `AgentFacade` 优先使用 Spring AI `ChatClient` + Tool Calling。
- 旧的自定义 LLM client、prompt registry 和 `TicketAgentOrchestrator` 已移除。
- 模型不可用时，回退到同一套 Tool、Guard、参数抽取和会话上下文，不再维护第二套 Agent 编排。

### rag

- `SpringAiEmbeddingModelClient` 优先调用 Spring AI `EmbeddingModel`。
- `EmbeddingService` 在启用 PGvector 后写入 Spring AI `VectorStore`。
- `RetrievalService` 在启用 PGvector 后优先使用 Spring AI `VectorStore` 检索。
- 默认保留 MySQL JSON 向量和本地 hash embedding 兜底。

### biz

`biz` 不调用 Spring AI。业务规则、权限、幂等、状态机和知识生产都必须保留在 `biz`。
