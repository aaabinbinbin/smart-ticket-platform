# Spring AI Integration Change 1

更新时间：2026-04-19

## 1. 本次变更目标

本次只接入 Spring AI 基础依赖和配置骨架，不重写工单业务逻辑。

目标是为后续迁移做好落点：

- Agent 对话模型调用迁移到 Spring AI `ChatClient`
- RAG 向量化迁移到 Spring AI `EmbeddingModel`
- RAG 检索迁移到 Spring AI PGvector `VectorStore`

## 2. 依赖说明

根 `pom.xml` 新增：

```xml
<spring-ai.version>1.0.5</spring-ai.version>
```

并通过 Spring AI BOM 管理版本：

```xml
<dependency>
    <groupId>org.springframework.ai</groupId>
    <artifactId>spring-ai-bom</artifactId>
    <version>${spring-ai.version}</version>
    <type>pom</type>
    <scope>import</scope>
</dependency>
```

同时将 Spring Boot 升级到 `3.4.13`，用于匹配 Spring AI 1.0.x 稳定版本线。

`smart-ticket-infra` 新增依赖：

- `spring-ai-starter-model-openai`：提供 Chat model 与 Embedding model 自动装配。
- `spring-ai-starter-vector-store-pgvector`：提供 PGvector VectorStore 自动装配。
- `spring-boot-starter-jdbc`：PGvector VectorStore 需要 JDBC 能力。

未引入 LangChain4j。

## 3. 配置说明

`smart-ticket-app/src/main/resources/application.yml` 新增 Spring AI 配置骨架：

```yaml
spring:
  ai:
    model:
      chat: ${SPRING_AI_MODEL_CHAT:none}
      embedding: ${SPRING_AI_MODEL_EMBEDDING:none}
    openai:
      api-key: ${OPENAI_API_KEY:}
      base-url: ${OPENAI_BASE_URL:https://api.openai.com}
      chat:
        options:
          model: ${OPENAI_CHAT_MODEL:gpt-4o-mini}
      embedding:
        options:
          model: ${OPENAI_EMBEDDING_MODEL:text-embedding-3-small}
    vectorstore:
      pgvector:
        enabled: ${SMART_TICKET_AI_VECTOR_STORE_ENABLED:false}
        initialize-schema: false
        schema-name: ${PGVECTOR_SCHEMA:public}
        table-name: ${PGVECTOR_TABLE:vector_store}
        dimensions: ${PGVECTOR_DIMENSIONS:1536}
```

默认全部关闭：

- `SMART_TICKET_AI_CHAT_ENABLED=false`
- `SMART_TICKET_AI_EMBEDDING_ENABLED=false`
- `SMART_TICKET_AI_VECTOR_STORE_ENABLED=false`
- `SPRING_AI_MODEL_CHAT=none`
- `SPRING_AI_MODEL_EMBEDDING=none`

原因：

- 当前业务主数据源仍是 MySQL。
- PGvector 需要 PostgreSQL + pgvector 扩展。
- 本阶段只建立配置骨架，不要求本地必须有模型密钥和向量库。

启用 OpenAI Chat 时，需要同时设置：

```text
SMART_TICKET_AI_CHAT_ENABLED=true
SPRING_AI_MODEL_CHAT=openai
OPENAI_API_KEY=<your-api-key>
```

启用 OpenAI Embedding 时，需要同时设置：

```text
SMART_TICKET_AI_EMBEDDING_ENABLED=true
SPRING_AI_MODEL_EMBEDDING=openai
OPENAI_API_KEY=<your-api-key>
```

项目自有开关位于：

```yaml
smart-ticket:
  ai:
    chat:
      enabled: false
    embedding:
      enabled: false
    vector-store:
      enabled: false
    pgvector:
      url: jdbc:postgresql://127.0.0.1:5432/smart_ticket_vector
      username: postgres
      password: postgres
      driver-class-name: org.postgresql.Driver
```

后续真正启用 PGvector 时，需要补齐 PostgreSQL 数据源隔离方案，避免和当前 MySQL 主业务库混用。

## 4. 模块落位说明

### `smart-ticket-infra`

新增：

- `com.smartticket.infra.ai.ChatClientConfig`
- `com.smartticket.infra.ai.EmbeddingModelConfig`
- `com.smartticket.infra.ai.VectorStoreConfig`

职责：

- 统一承接 Spring AI Chat、Embedding、VectorStore 的外部能力适配。
- 不包含工单业务规则。
- 不依赖 `biz`、`agent`、`rag`。

### `smart-ticket-agent`

本次删除原手写 HTTP 模型客户端：

- `OpenAiCompatibleLlmClient`

新增：

- `SpringAiLlmClient`

职责：

- 使用 Spring AI `ChatClient` 完成模型文本调用。
- 保留原 `LlmClient` 抽象，避免改动 Agent 编排主流程。
- 不直接执行 Tool。
- 不判断业务合法性。

### `smart-ticket-rag`

本次暂不改造 RAG 执行链路。

后续迁移方向：

- 用 Spring AI `EmbeddingModel` 替换 `LocalHashEmbeddingModelClient`。
- 用 Spring AI `VectorStore` 替换应用内向量扫描。
- 保留 `RetrievalService` 作为项目检索用例入口。

### `smart-ticket-biz`

本次不修改。

以下能力仍必须留在 `biz`：

- 工单创建、分配、转派、状态流转、关闭
- 权限校验
- 幂等
- 操作日志
- 关闭后知识构建资格判断
- 标准化知识文本生成

## 5. 当前边界

Spring AI 现在只接管模型调用适配层的基础骨架。

它不接管：

- 工单业务规则
- 工单写操作
- 权限判断
- RAG 结果是否阻止创建工单
- 知识是否可以沉淀的业务判断
