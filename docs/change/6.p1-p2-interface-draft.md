# P1/P2 接口草案

更新时间：2026-04-20

本文档记录 P1/P2 接口规划。当前已实现工单组和工单队列接口；其余仍为草案。

## P1 接口草案

### 工单组

- `POST /api/ticket-groups`
  - 创建工单组。
  - 请求字段：`groupName`、`groupCode`、`ownerUserId`、`enabled`。
  - 当前状态：已实现。
- `GET /api/ticket-groups`
  - 分页查询工单组。
  - 查询字段：`keyword`、`enabled`、`pageNo`、`pageSize`。
  - 当前状态：已实现。
- `GET /api/ticket-groups/{groupId}`
  - 查询工单组详情。
  - 当前状态：已实现。
- `PUT /api/ticket-groups/{groupId}`
  - 更新工单组名称、负责人和启停状态，不修改 `groupCode`。
  - 当前状态：已实现。
- `PATCH /api/ticket-groups/{groupId}/enabled`
  - 启用或停用工单组。
  - 当前状态：已实现。

### 工单队列

- `POST /api/ticket-queues`
  - 创建工单队列。
  - 请求字段：`queueName`、`queueCode`、`groupId`、`enabled`。
  - 当前状态：已实现。
- `GET /api/ticket-queues`
  - 分页查询队列。
  - 查询字段：`groupId`、`keyword`、`enabled`、`pageNo`、`pageSize`。
  - 当前状态：已实现。
- `GET /api/ticket-queues/{queueId}`
  - 查询队列详情。
  - 当前状态：已实现。
- `PUT /api/ticket-queues/{queueId}`
  - 更新队列名称、所属工单组和启停状态，不修改 `queueCode`。
  - 当前状态：已实现。
- `PATCH /api/ticket-queues/{queueId}/enabled`
  - 启用或停用工单队列。
  - 当前状态：已实现。
- `PUT /api/tickets/{ticketId}/queue`
  - 将工单绑定到指定工单组和队列，不修改处理人和状态。
  - 当前状态：已实现。

### SLA

- `POST /api/ticket-sla-policies`
  - 创建 SLA 策略。
  - 请求字段：`policyName`、`category`、`priority`、`firstResponseMinutes`、`resolveMinutes`。
  - 当前状态：已实现。
- `GET /api/ticket-sla-policies`
  - 分页查询 SLA 策略。
  - 当前状态：已实现。
- `GET /api/ticket-sla-policies/{policyId}`
  - 查询 SLA 策略详情。
  - 当前状态：已实现。
- `PUT /api/ticket-sla-policies/{policyId}`
  - 更新 SLA 策略。
  - 当前状态：已实现。
- `PATCH /api/ticket-sla-policies/{policyId}/enabled`
  - 启用或停用 SLA 策略。
  - 当前状态：已实现。
- `GET /api/tickets/{ticketId}/sla`
  - 查询某个工单的 SLA 实例。
  - 当前状态：已实现。

### 自动分派规则

- `POST /api/ticket-assignment-rules`
  - 创建分派规则。
  - 请求字段：`ruleName`、`category`、`priority`、`targetGroupId`、`targetQueueId`、`targetUserId`、`weight`。
  - 当前状态：已实现。
- `GET /api/ticket-assignment-rules`
  - 分页查询分派规则。
  - 当前状态：已实现。
- `GET /api/ticket-assignment-rules/{ruleId}`
  - 查询分派规则详情。
  - 当前状态：已实现。
- `PUT /api/ticket-assignment-rules/{ruleId}`
  - 更新分派规则。
  - 当前状态：已实现。
- `PATCH /api/ticket-assignment-rules/{ruleId}/enabled`
  - 启用或停用分派规则。
  - 当前状态：已实现。
- `POST /api/tickets/{ticketId}/assignment-preview`
  - 预览分派结果，不执行写操作。
  - 当前状态：已实现。
- `POST /api/tickets/{ticketId}/auto-assign`
  - 按命中的自动分派规则执行真实分派。
  - 当前状态：已实现最小闭环，支持规则配置 `targetUserId` 时真实分派，并支持同步绑定目标队列。

## P2 接口草案

### 审批

- `POST /api/tickets/{ticketId}/approval-tasks`
  - 创建审批任务。
- `POST /api/ticket-approval-tasks/{taskId}/approve`
  - 审批通过。
- `POST /api/ticket-approval-tasks/{taskId}/reject`
  - 审批拒绝。

### 子任务

- `POST /api/tickets/{ticketId}/subtasks`
  - 创建子任务。
- `GET /api/tickets/{ticketId}/subtasks`
  - 查询工单子任务。
- `PATCH /api/ticket-subtasks/{subtaskId}/status`
  - 更新子任务状态。

### 知识治理

- `POST /api/ticket-knowledge/{knowledgeId}/reviews`
  - 发起知识审核。
- `POST /api/ticket-knowledge-reviews/{reviewId}/approve`
  - 审核通过。
- `POST /api/ticket-knowledge-reviews/{reviewId}/reject`
  - 审核拒绝。
- `GET /api/ticket-knowledge/{knowledgeId}/versions`
  - 查询知识版本。

### 指标与租户

- `GET /api/ticket-metrics/daily`
  - 查询日指标。
- `POST /api/tenants`
  - 创建租户。
- `GET /api/tenants`
  - 查询租户列表。

## 共同约束

- 写接口必须经过 biz service，不允许 controller 或 agent 直接访问 mapper。
- P1/P2 接口落地前必须补权限点、审计日志和幂等策略。
- 涉及 Agent 的接口必须先定义 Tool 风险等级，再决定是否暴露给模型。
- 当前 SLA 仅生成实例和截止时间，不包含违约扫描、升级和通知。
- 当前真实自动分派必须命中目标处理人；如果规则同时命中目标队列，会先完成工单队列绑定。
