# P1 SLA Policy And Instance

更新时间：2026-04-20

## 本次实现范围

本次实现 P1 的 SLA 基础能力：

- SLA 策略 `ticket_sla_policy`
- 工单 SLA 实例 `ticket_sla_instance`
- SLA 策略管理 API
- 工单 SLA 实例查询 API
- 工单创建后生成 SLA 实例
- 工单分配后刷新 SLA 实例

不实现：

- 定时违约扫描。
- SLA 升级策略。
- 自动通知。
- Agent 自动操作 SLA。

## 策略匹配规则

当前按工单分类和优先级匹配启用策略：

- `category` 为空表示通配分类。
- `priority` 为空表示通配优先级。
- 同时命中多条时，优先选择更具体的策略。
- 具体度相同时，选择 ID 较大的策略。

## 截止时间计算

当前以工单创建时间作为基准：

- `firstResponseDeadline = ticket.createdAt + firstResponseMinutes`
- `resolveDeadline = ticket.createdAt + resolveMinutes`

如果工单创建时间为空，则以当前时间兜底。

## 边界

- SLA 写操作必须由 ADMIN 执行。
- 查询工单 SLA 实例前先复用工单详情权限判断。
- SLA 不替代工单状态机。
- SLA 不直接触发自动分派或关闭工单。
## 2026-04-20 违约扫描补充

已补充手动 SLA 违约扫描能力：

- `POST /api/ticket-sla-instances/breach-scan?limit=100`
- 仅 `ADMIN` 可操作。
- 扫描 `breached = 0` 且 `resolve_deadline <= scanTime` 的 SLA 实例。
- 成功命中的实例会被标记为 `breached = 1`。

仍不实现：

- 定时调度。
- SLA 升级策略。
- 自动通知。
- Agent 自动操作 SLA。
