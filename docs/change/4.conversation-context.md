# Conversation Context Change 4

更新时间：2026-04-20

## 目标

在 `smart-ticket-agent` 中实现短会话上下文，并与 `AgentFacade` 集成。

第一版只做短上下文，不做长期记忆，不做复杂实体图谱推断。

## Redis Key

```text
agent:session:{sessionId}
```

## 会话字段

`AgentSessionContext` 至少维护：

- `activeTicketId`
- `activeAssigneeId`
- `lastIntent`
- `recentMessages`

## 执行链路

`AgentFacade` 当前链路：

1. 调用前通过 `AgentSessionService.load(sessionId)` 加载上下文。
2. 使用 `IntentRouter` 结合上下文判断本轮意图。
3. Spring AI Tool Calling 或确定性兜底链路执行 Tool。
4. 执行前通过 `AgentSessionService.resolveReferences(...)` 做最小指代消解。
5. 执行后通过 `AgentSessionService.updateAfterTool(...)` 更新上下文。

## 最小指代消解

当前支持：

- “刚才那个工单”
- “它”
- “这个问题”

指代只映射到短上下文中的 `activeTicketId` 或 `activeAssigneeId`。如果上下文没有对应实体，不做猜测，由 Tool 参数校验返回缺参提示。

## 当前边界

- 不做长期记忆。
- 不保存完整对话历史。
- 不让上下文绕过 biz 权限判断。
- 不依赖旧 `TicketAgentOrchestrator`。
