# Agent Chat API Change 2

更新时间：2026-04-20

## 接口说明

`smart-ticket-api` 提供 Agent 对话入口：

```http
POST /api/agent/chat
Authorization: Bearer <jwt>
Content-Type: application/json
```

请求模型：`AgentChatRequest`

```json
{
  "sessionId": "s-001",
  "message": "查询 100 号工单"
}
```

响应模型：`AgentChatResponse`

```json
{
  "sessionId": "s-001",
  "intent": "QUERY_TICKET",
  "reply": "工单查询结果...",
  "result": {},
  "route": {},
  "context": {},
  "springAiChatReady": false
}
```

## Agent 入口链路

当前 `AgentFacade` 的执行顺序：

1. 加载 Redis 短会话上下文。
2. `IntentRouter` 判断本轮意图。
3. 如果 Spring AI `ChatClient` 可用，调用 Spring AI Tool Calling。
4. 如果模型不可用、未触发 Tool 或调用失败，执行确定性兜底链路。
5. Tool 执行后更新 Redis 短会话上下文。
6. 返回统一 `AgentChatResult`，由 api 模块转换为 HTTP 响应。

## 支持的意图

第一版只支持四类意图：

- `QUERY_TICKET`
- `CREATE_TICKET`
- `TRANSFER_TICKET`
- `SEARCH_HISTORY`

## 当前边界

- 不做复杂多 Agent 编排。
- 不引入 MCP。
- 不把所有意图塞进一个超级 prompt。
- 不再使用旧的 `TicketAgentOrchestrator` 或自定义 LLM client。
- 写操作只能通过 Tool 调用 `biz` service。
