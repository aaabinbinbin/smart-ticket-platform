# P1 SLA Breach Scan

更新时间：2026-04-20

## 本次实现范围

本次只实现 P1 最小闭环的 SLA 违约扫描：

- 从 `ticket_sla_instance` 中扫描 `breached = 0` 且 `resolve_deadline <= 当前扫描时间` 的实例。
- 按 `resolve_deadline` 升序处理，单次默认 `100` 条，最大 `1000` 条。
- 将命中的实例标记为 `breached = 1`。
- 返回本次扫描时间、扫描上限、候选数量、成功标记数量和已标记实例 ID。

## 接口

- `POST /api/ticket-sla-instances/breach-scan?limit=100`
- 权限：`ADMIN`
- 响应字段：
  - `scanTime`：本次扫描使用的业务时间
  - `limit`：本次扫描上限
  - `candidateCount`：候选违约实例数量
  - `markedCount`：成功标记数量
  - `breachedInstanceIds`：成功标记的 SLA 实例 ID

## 模块落位

- `domain`
  - `TicketSlaInstanceMapper.findBreachedCandidates`
  - `TicketSlaInstanceMapper.markBreached`
- `biz`
  - `TicketSlaInstanceRepository`
  - `TicketSlaService.scanBreachedInstances`
  - `TicketSlaScanResultDTO`
- `api`
  - `TicketSlaController`
  - `TicketSlaScanResultVO`

## 边界

- 扫描只更新 SLA 实例的 `breached` 标记。
- 不修改工单状态。
- 不做升级策略。
- 不发送通知。
- 不自动分派或转派工单。
- 不接入 Agent Tool，避免模型触发批量写操作。
