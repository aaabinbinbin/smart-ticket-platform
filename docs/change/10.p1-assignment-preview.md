# P1 Assignment Preview

更新时间：2026-04-20

## 本次实现范围

本次实现 P1 自动分派的 preview 能力，并补充真实自动分派的最小闭环：

- 自动分派规则 `ticket_assignment_rule`
- 自动分派规则管理 API
- 工单自动分派 preview API
- 工单真实自动分派 API
- 命中目标队列时同步绑定工单队列

不实现：

- 仅命中工单组或队列但没有目标处理人时的真实分派。
- Agent 自动执行分派。

## 匹配规则

preview 根据工单分类和优先级匹配启用规则：

- `category` 为空表示通配分类。
- `priority` 为空表示通配优先级。
- 优先选择 `weight` 更大的规则。
- 权重相同选择更具体的规则。
- 仍相同选择 ID 更大的规则。

## 规则目标校验

创建或更新规则时校验：

- 至少存在一个目标：`targetGroupId`、`targetQueueId`、`targetUserId`。
- `targetGroupId` 必须指向启用的工单组。
- `targetQueueId` 必须指向启用的工单队列。
- 同时配置工单组和队列时，队列必须属于该工单组。
- `targetUserId` 必须是启用用户且具备 STAFF 角色。

## 接口

- `POST /api/ticket-assignment-rules`
- `PUT /api/ticket-assignment-rules/{ruleId}`
- `PATCH /api/ticket-assignment-rules/{ruleId}/enabled`
- `GET /api/ticket-assignment-rules/{ruleId}`
- `GET /api/ticket-assignment-rules`
- `POST /api/tickets/{ticketId}/assignment-preview`
- `POST /api/tickets/{ticketId}/auto-assign`

## 边界

- 规则写操作必须由 ADMIN 执行。
- preview 前会复用工单详情权限判断。
- preview 不写工单、不写操作日志、不调用真实分派。
- `auto-assign` 只在命中规则明确配置 `targetUserId` 时执行。
- `auto-assign` 命中 `targetQueueId` 时，会先调用工单队列绑定，再调用真实分派。
- `auto-assign` 复用现有工单分配服务，不能绕过状态机和权限。
- 仅命中工单组或队列且没有目标处理人时，当前不执行真实分派。
